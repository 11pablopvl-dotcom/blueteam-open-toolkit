# Contributing to the Blue Team Open Toolkit

Thanks for helping make this the go-to **free** resource for CrowdStrike Falcon
blue teamers. Contributions are welcome and reviewed quickly.

## What we accept

- New high-quality **free** resources (official docs, open-source tools, labs, blogs).
- Fixes to broken links, outdated info, or technical errors.
- Improvements to the checklists and FQL samples — clearer, more accurate, better tuned.

## Ground rules

- **Vendor-honest.** No marketing fluff. If a tool has caveats, say so.
- **Practitioner-useful.** Would it help someone on shift at 2 a.m.? If not, skip it.
- **No exam dumps, no NDA-violating content.** We respect CrowdStrike's certification
  terms — material here is based only on publicly documented objectives.
- **Test before you claim.** Any detection or FQL must be tested; note the
  false-positive cost where relevant.

## How to contribute

1. Fork the repo and create a branch: `git checkout -b add-<short-description>`.
2. Make your change, keeping formatting consistent with the existing files.
3. Open a Pull Request describing what you added and why it's useful.

Small fixes (typos, dead links) can go straight to a PR — no issue needed.

## Style

- Links: `[Name](url) — one-line why it matters`.
- Keep it concise. Dense and useful beats long and padded.
